# 小児用量計算アプリ (Pediatric Calc) 仕様書

## 1. 概要 (Overview)
本アプリケーションは、小児の薬用量計算を支援するためのWebツールです。
薬剤師や医師が処方監査や処方設計を行う際に、患児の年齢・体重に基づき、添付文書に記載された用法用量を即座に算出・確認し、安全かつ迅速な医療の提供に寄与することを目的としています。
当初は小児服薬支援アプリ「おくすりぱっくん」の一部として検討されましたが、成分ベースでの計算というデータ構造の違いや、医療従事者向けというターゲットユーザーの違いから、独立したアプリケーションとして構築・運用されています。

## 2. 機能要件 (Functional Requirements)

### 2.1 入力機能 (患者情報)
* **ダイヤルピッカーによる年齢・体重入力**: 
  モバイル端末での操作性を重視し、直感的に選択可能なスクロール式ダイヤルピッカーを採用。
  * **年齢入力**: 「歳」と「ヶ月」を別々に設定可能。
  * **体重入力**: 1〜100までのダイヤルを設定。誤操作を防止するため小数点以下の入力は廃止。
* **標準体重の自動補完**: 
  入力された年齢から推計される標準体重をワンタップで反映する機能を搭載。

### 2.2 薬剤選択・管理機能
* **カテゴリ別タブ切り替え**: 
  テキスト検索を廃止し、薬効群（抗生剤、抗ウイルス、漢方薬など）ごとにタブで切り替えるグリッド表示を採用。素早いアクセスを実現。
* **処方シート（複数選択）**: 
  複数の薬剤を同時に「処方シート」に追加し、総量や組み合わせを並列で確認可能。
* **薬剤の非選択（削除）**: 
  処方シート上のゴミ箱アイコンを操作することで、選択済みリストから当該薬剤を直感的に削除できる。
* **規格・剤形の選択**: 
  同一成分で複数規格（例: 10%細粒と20%細粒）が存在する場合、選択時にプルダウン等で指定可能。
* **パーソナライズ (Local Storage保存)**: 
  選択中の薬剤リストはブラウザのLocalStorageに自動保存される。次回アクセス時も状態が復元されるため、「My医薬品集」として活用可能。

### 2.3 計算・表示機能
* **リアルタイム用量計算**: 
  体重・年齢の変更に連動して、1日量および1回量を即座に再計算。
* **上限値・下限値チェック**: 
  計算結果が添付文書の推奨範囲（mg/kg/日）に収まっているかを判定し、1日最大量や成人量を超過する場合は適切にリミットをかける。
* **特殊計算ロジック対応**: 
  * **Augsberger式・Crawford式等**: 体重だけでなく年齢を加味した小児用量計算。
  * **漢方の年齢基準計算**: 年齢区分に応じた分量割合の適用。
  * **体重/年齢別固定量**: ゾフルーザ（体重区分）やイナビル（年齢区分）など。
* **計算根拠バッジとアイコン**: 
  各薬剤カードのタイトル付近に、年齢基準（Age）または体重基準（Weight）のどちらで計算されたかを示すバッジやアイコンを配置し、根拠を明確化。
* **詳細情報モーダル**: 
  薬剤カードクリックによりモーダルを展開し、計算方法（Augsberger式など）や使用されたパラメータ詳細を表示。

### 2.4 PMDA添付文書連携機能
* **外部URLリンク機能**: 
  各薬剤の「Package Insert」リンクから、直接PMDAの当該添付文書ページへ遷移可能。
* **用法用量テキストの表示機能**: 
  PMDAのXMLデータからYJコードをキーにして抽出された「用法及び用量」のテキストを、アプリケーション内のモーダルなどで詳細として閲覧可能にする仕組みを実装。

## 3. 非機能要件 (Non-Functional Requirements)
* **モバイルファースト・レスポンシブデザイン**: 
  狭い画面でも情報の一覧性を担保するため余白を最小化。処方シートは横スクロールにより画面の占有を防止し、計算結果領域を最大化。
* **パフォーマンス**: 
  Vanilla JavaScriptにより軽量・高速に動作。状態管理にLocalStorageを活用し、ネットワーク通信やデータベースアクセスによる遅延を削減。
* **保守性・拡張性**: 
  UIコンポーネント、計算ロジック、データ連携を独立したモジュールとして管理。新規薬剤や計算式の追加を容易にするための構造化を実施。

## 4. 技術スタック (Tech Stack)
* **Frontend**: Vanilla JavaScript (ES Modules), HTML5, CSS3
* **Design**: Custom CSS (CSS Variables, CSS Grid/Flexbox), FontAwesome (Icons)
* **Data Processing**: PMDA SGML/XML Parsing (Node.js script for data extraction)
* **Storage**: Web Storage (Local Storage)

## 5. データ構造 / スキーマ (Data Structure / Schema)
アプリケーションのコアとなる薬剤データ（`PEDIATRIC_DRUGS`）の構造定義：

```javascript
{
  id: string,                 // 一意の識別子
  name: string,               // 表示名
  brandName: string,          // 先発品名や通称
  yjCode: string,             // 代表YJコード (PMDA連携のキー)
  piUrl: string,              // 添付文書のリンクURL
  potency: number,            // 力価（例: 10%細粒なら100mg/g）
  
  // 計算ロジックの定義
  dosage: {
    minMgKg: number,          // 最小用量 (mg/kg/日)
    maxMgKg: number,          // 最大用量 (mg/kg/日)
    timesPerDay: number,      // 1日の標準投与回数
    absoluteMaxMgPerDay: number, // 1日の絶対最大許容量 (成人量等)
    isByTime: boolean,        // 1回量を基準に計算するか否か
    timeMgKg: number,         // 1回あたりの用量 (mg/kg/回)
    // 疾患別、体重・年齢分岐用の関数やマッピングオブジェクトが含まれる場合あり
  },
  
  // 疾患別の用量定義 (任意)
  diseases: [
    { id: string, label: string, dosage: Object }
  ],
  
  // 用量規格のバリエーション (任意)
  subOptions: [
    { id: string, label: string, potency: number }
  ]
}
```

## 6. 画面遷移 / UI設計 (UI/UX Design)
1. **メイン画面**:
   上部に年齢・体重入力用のダイヤルピッカーを配置。直下に選択済み薬剤を表示する「処方シート」（水平スクロール可能）を配置。
2. **薬剤選択領域**:
   カテゴリタブで薬効を切り替え。グリッド配置された薬剤カードには、名称、代表的な用量、および「Age」「Weight」の計算基準バッジを表示。
3. **モーダル領域**:
   薬剤詳細を確認するためのモーダル。計算式の根拠（Augsberger式適用など）や、PMDAデータから取得した正確な用法・用量テキストを提示。

## 7. 外部連携 (External Integrations)
* **PMDAオープンデータ**: 
  提供される2020年代以降のSGML/XMLデータをパースし、各薬剤のYJコードと紐付けて「用法及び用量」の情報を抽出するバッチ処理と連携。抽出データはJSON等の形式でアプリに供給される。

## 8. 制限事項 / 特記事項 (Constraints / Notes)
* **免責事項**: 
  本アプリケーションは医療従事者向けの計算「補助」ツールです。出力される結果から直接処方意図を決定するものではなく、最終的な投与量や適応の判断は医師・薬剤師の責任のもとで行うものとします。
* **データ保証**: 
  本ツールの薬剤基礎データは手書きやスクリプト抽出のハイブリッドで構築されています。LLM等による自動生成データの誤りを排除するため、定期的な目視確認と添付文書との突合を行っていますが、使用時は必ず添付文書等の一次文書をご確認ください。
