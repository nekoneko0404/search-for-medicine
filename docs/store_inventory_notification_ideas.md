# search-for-medicine 医薬品供給状況：店舗在庫（2000品目）一括登録・通知機能の検討

店舗が抱える約2000品目の在庫（YJコード等）を一括登録し、その中から「通常出荷」等へのステータス変化があったものだけを抽出・通知する機能についての技術的難易度および価格設定の検討レポートです。

---

## 1. 機能の実現可能性と技術的難易度

結論から言うと、**技術的に十分に実現可能であり、難易度は「中〜やや高」**です。
現在のアーキテクチャ（定期的なCSVのスクレイピング・解析）をベースに、データベース駆動の通知システムを追加拡張する形になります。

### 必要な機能コンポーネント

1.  **一括登録インターフェース（フロントエンド）**
    *   YJコード（あるいはレセコンから出力された厚労省コード・JANコード等）のリストをテキストエリアに「コピペ」または「CSVでアップロード」するUI。
    *   *難易度：低*（JavaScriptで容易にパース可能）。
2.  **ユーザー別・店舗別の監視リストDB（バックエンド）**
    *   「どのユーザー（店舗）が」「どのコードを」監視しているかを保存するデータベース（Cloudflare D1やFirebase Firestore）。2000品目 × 数十〜数百店舗であれば、データ量は非常に軽量です。
    *   *難易度：低〜中*。
3.  **差分検知＆通知エンジン（バックエンド Worker）**
    *   日次（または定期）で実行されるスクリプトが最新の供給状況データ（MHLWのCSV等）を取得した際、**「前回の状態」と「今回の状態」を比較し、ステータスが変化した薬剤（かつ、それが誰かの監視リストに含まれているか）を割り出す**バッチ処理。
    *   変化があったユーザーに対して、メール（SendGrid等）またはLINE/Push通知を送信する。
    *   *難易度：中〜高*（状態の履歴（スナップショット）を保持し、効率よく差分を計算（Diff）するロジックが必要になります）。

### 想定される技術的課題（落とし穴）

*   **コードの揺れ:** 店舗のレセコンから出力されるコードが、システム内部で使っている「YJコード」と完全に一致するか（厚労省コード、レセプト電算コードとの紐付け・変換テーブルが別途必要になる可能性が高い）。
*   **誤検知・通知スパム:** 出荷元（メーカー）側のデータ修正等で、実質的な供給状況が変わっていないのにステータス文言だけが変わって大量に通知が飛ぶリスク。差分判定ロジックを厳密にする必要があります。

## 2. B2B（B2Pharmacy）価格設定の妥当性

この機能は、薬局の「日々の発注業務（出荷調整品目かどうかの確認、復活した品目のいち早い確保）」という**明確なコスト（人件費・時間）を削減**する、非常に強力な業務支援ツールです。一般消費者向けアプリとは異なり、B2B SaaSとしての価格設定が基本となります。

### ① サブスクリプション価格（月額）の考察

*   **最低ライン (月額 1,000円 〜 1,980円 / 1店舗)**
    *   零細薬局でも稟議や決裁なしに、管理薬剤師の権限（あるいは個人のポケットマネー）で「とりあえず試してみよう」と思える価格帯。
    *   まずは初期ユーザーを獲得し、フィードバックを得てシステムを安定させるフェーズ（ベータ版〜初期リリース）に最適。
*   **適正価格帯 (月額 3,000円 〜 5,000円 / 1店舗)**
    *   この機能により、毎日30分のアナログな在庫確認・発注作業（薬剤師・事務員の人件費：時給2,000円換算で月額約10,000円相当）が削減され、かつ「品薄薬を他店より早く確保できる」という経済的メリット（機会損失の防止）を提供できます。
    *   ROI（投資対効果）を考えれば、月額3,000円〜5,000円は薬局経営者にとって十分に安く、契約を継続する強力な動機になります。
*   **高価格帯 (月額 10,000円〜 / 複数店舗を持つ法人向け)**
    *   チェーン薬局（本部）が5〜10店舗分の在庫を統合管理し、ダッシュボード機能（CSVでの差分一括ダウンロードなど）を利用できる「エンタープライズ（法人包括）プラン」。

### ② 初期費用（セットアップ費）の考察

*   レセコンからの在庫リスト出力手順のサポートや、**初回データのクレンジング（コードの紐付けエラー解消など）**が必要になるケースが想定されます。
*   必要に応じて「初期導入サポート費（5,000円〜10,000円）」を設定するか、または「最初の1ヶ月は無料（無料期間中にリストをアップロードして効果を体感してもらう）」とするのがSaaSの定石です。

## 3. 推奨される事業展開ステップ

1.  **MVP（Minimum Viable Product）開発:**
    *   まずは「CSVやコピペによる登録機能（最大数限定）」と、「ステータスが変化したらWeb画面上でハイライトされる（メール通知はまだしない）」機能だけを作り、知り合いの店舗等でテスト運用する。
2.  **通知インフラの実装:**
    *   MVPで差分検知のロジックが安定したら、メール送信（または専用LINEボット経由）での通知機能を実装。
3.  **β版リリース（月額1,500円等の低価格）:**
    *   初期ユーザーのフィードバックをもとに、どのステータス変化（例：「出荷停止」→「限定出荷」よりも「限定出荷」→「通常出荷」の通知だけ欲しい、等）が一番求められているかUIを洗練させる。
4.  **本番リリース（月額3,000円〜）:**
    *   複数店舗の一括管理機能等を追加し、適正価格でマーケティングを行う。

**結論:** この機能は薬局のペイン（痛み・課題）を直接解決できるため、月額3,000円〜5,000円/店舗の価値が十分にあります。技術的にも現在の資産を活かして構築可能な、最も確実性の高いマネタイズの柱になり得ます。
