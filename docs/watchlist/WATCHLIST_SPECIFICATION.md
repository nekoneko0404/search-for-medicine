# 店舗採用薬 供給監視機能 仕様書

## 機能概要
自店舗の採用薬（YJコードで登録）の供給状況を監視し、同一成分・規格（YJコード上9桁）内の供給バランスを視覚的に把握できるダッシュボードです。

## UI・レイアウト
「情報更新表示」ページのデザインに基づいたテーブルレイアウトを採用しています。

- **ダッシュボード**: 
    - **メトリクスカード**: 「総採用薬数」「改善（制限解除）」「要注意（悪化）」「最短解消見込み」の4つの指標をリアルタイム表示。
    - **供給状況グラフ**: 通常/限定/停止の各比率をプログレスバーで可視化。
- **フィルタエリア**: 
    - 製品名/成分名/YJコードによる検索。
    - 出荷区分（通常/限定/停止）による絞り込み。
    - 区分（内/注/外）による絞り込み。
    - 特殊フィルタ（採用薬のみ、ステータス変化あり、制限解除）。
- **メインテーブル**:
    - **品名**: 製品名。クリックで詳細表示。
    - **成分名**: 有効成分名。
    - **出荷状況**: **個別品目のステータスボタン**と、**同一規格内（9桁YJ一致）の供給比率バー**を併記。周辺状況と個別の状況を同時に確認可能。
    - **制限理由/解消/時期/出荷量/更新日**: PMDAデータに基づく詳細情報。各カラム名をクリックすることでソートが可能。
- **更新期間フィルタ**: 「7日以内」「30日以内」等の期間で絞り込みが可能。

## 技術仕様
- **周辺品目の集計ロジック**:
    - YJコードの上9桁を取得し、その9桁で始まる全ての医薬品の出荷ステータスを集計します。
    - これにより、ある製品が限定出荷でも、同一規格の他社製品は通常出荷であるといった「代替品の有無」を直感的に判断可能にします。
- **データ永続化**: 採用薬リスト（YJコード）は `localStorage` に保存され、ページを閉じても維持されます。
- **検索ロジック**: 包含キーワードに加え、ハイフンや長音（`-`, `ー`, `－`）を接頭辞とした除外キーワードに対応。
- **ユニットテスト**: `vitest` を用いて集計ロジックやフィルタ処理のテストを実施。

## 収益化・制限仕様 (Milestone 3 実装済み)

現在は **Testing in progress** 期間として、以下の制限・プラン管理を行っています。

### 1. プランと監視上限 (usageLimit)
- **Freeプラン**: 監視上限 20品目。
- **Standardプラン**: 監視上限 3,000品目。
- **制限ロジック**: `batch` API 実行時に、DBに設定された `usage_limit` を超えるコードは切り捨てられます。
- **UI表示**: 監視リスト登録画面で「現在の登録数 / 上限数」を表示し、超過時は赤字で警告します。

### 1. サーバーサイド・データ管理（Cloudflare Worker / D1）
- **DB駆動の監視**: YJコード(約2000品目)をサーバー側DBで管理。
- **高精度差分検知**: 日次更新データと前回のスナップショットをバックエンドで自動比較。

### 2. 自動通知インフラ
- **即時通知**: ステータス変化（特に「限定出荷」→「通常出荷」の復帰）を検知し、LINEまたはメールでプッシュ通知。
- **フィルタリング通知**: ユーザーが設定した特定の条件に合致する場合のみ通知。

### 3. データ連携・一括登録
- **レセコン連携の簡易化**: 既存のコピペ・CSV登録を強化し、各社レセコンからの出力形式に対応したクレンジング機能の搭載。

## 技術的決定事項
- **TypeScript**: 堅牢な型定義。
- **Logic抽出**: 単体テストを容易にするため、UI非依存のロジックを `logic.ts` に分離。
- **正規化**: 日本語文字の揺れを吸収する `normalizeString` の徹底利用。
- **Backend Architecture**: Cloudflare Workers + D1 によるエッジコンピューティング（ステートレスなフロントエンドからの脱却）。
