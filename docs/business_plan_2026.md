# 事業計画書 2026 (Search for Medicine)

## 1. 事業の目的
医薬品の供給不足という社会課題に対し、デジタル技術を活用して薬剤師・調剤事務の業務負荷を軽減し、患者への安定した医薬品供給をサポートします。

## 2. ターゲットセグメントと提供価値

| ターゲット | 提供価値 (Value Proposition) | 主要機能 |
| :--- | :--- | :--- |
| **B2B (薬局・医療機関)** | 在庫管理の効率化、欠品リスクの早期検知、業務のデジタル化。 | 医薬品供給状況アラート（店舗在庫監視）、一括インポート、CSV連携。 |
| **B2C (個人薬剤師・一般)** | 臨床・調剤業務の即時サポート、モバイルでの機動力。 | 小児・腎機能計算、全DI検索機能、スマホアプリ。 |

## 3. 収益モデルと価格設定

### 3.1. B2B サブスクリプション
- **内容:** 施設単位での包括ライセンス
- **価格:** 5,000円 / 月（税込・想定）
- **決済方法:** 銀行振込（請求書払い）または Stripe パスコード認証
- **目標:** 2026年6月開始、20契約達成

### 3.2. B2C サブスクリプション (スマホアプリ)
- **内容:** 個人向けの全機能利用ライセンス
- **価格:** 300円 / 月（税込・想定）
- **決済方法:** Google Play / App Store アプリ内課金 (RevenueCat)
- **目標:** 2026年8月開始、100ユーザー達成

## 5. ロードマップ 2026

### Phase 1: B2B MVP リリース (〜6月)
- **機能開発:** 医薬品供給アラート（店舗在庫監視）機能の実装。
- **認証基盤:** B2Bパスコード認証モデルの実装。
- **営業活動:** 近隣の薬局やチェーンへのテスト導入・フィードバック収集。

### Phase 2: モバイルアプリ化とB2C展開 (〜8月)
- **技術検証:** Capacitorによるネイティブアプリ化 (iOS/Android)。
- **課金実装:** RevenueCatを用いたアプリ内課金システムの構築。
- **リリース:** ストア公開と個人ユーザー向けマーケティング開始。

## 6. 技術戦略
- **サーバーレス:** Cloudflare Pages/Workers/D1を活用し、低コストでスケーラブルな基盤を維持。
- **プライバシー第一:** 個人情報を極力保持しない（匿名ID活用）設計を継続。
