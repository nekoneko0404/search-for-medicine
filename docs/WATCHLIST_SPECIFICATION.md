# 店舗採用薬 供給監視（Watchlist）機能 仕様書

## 1. 概要
本機能は、薬局の店舗採用品（在庫品）の供給状況を効率的に監視・把握するためのダッシュボードです。数万件に及ぶ全国の供給停止・制限情報の中から、自店舗に関連する品目のみを自動的に抽出し、変化を即座に視認できるように設計されています。

### 開発の背景
- 医薬品の供給不安定が長期化する中、薬剤師は日々膨大なリストから自店舗の採用薬を探し出す作業に追われている。
- 従来の検索機能では「品目名」での検索が主であり、多品目を一括で監視する仕組みが不足していた。

## 2. 対象ユーザー
- 保険薬局の薬剤師
- 薬局の在庫管理担当者

## 3. 主要機能

### 3.1 店舗採用薬の登録（ウォッチリスト）
- **YJコードによる登録**: レセコンや在庫管理システムから出力したYJコード（9桁または12桁）を改行区切りで一括登録可能。
- **最大登録件数**: 4,000件（一般的な薬局の採用数をカバー）。
- **永続化**: ブラウザの `localStorage` を使用して保存。

### 3.2 ダッシュボード表示
- **サマリービュー（マガジンスタイル）**: 成分ごとにグルーピングし、成分内の供給バランス（通常/限定/停止）をバーグラフで表示。
- **詳細ビュー**: 選択した成分に含まれる各製品（メーカー別）の具体的なステータス、制限理由、解消見込みを表示。

### 3.3 変化検知（スナップショット機能）
- **状態の記録**: 「現在の状態を記録」ボタンにより、現在の全品目のステータスをベースラインとして保存。
- **差分表示**: 次回データ更新時に、前回記録時からの変化（例：通常→限定、限定→通常）をバッジで強調表示。
- **フィルタリング**: 「変化あり」「制限解除」のみを抽出するクイックフィルターを搭載。

### 3.4 検索・フィルタリング
- **クイック検索**: 製品名、成分名、YJコードのAND/OR検索（マイナス検索対応）。
- **出荷区分フィルター**: 通常出荷、限定出荷、供給停止の表示切り替え。

## 4. 技術選定理由
- **Vanilla TS (Vite)**: 高速な動作とメンテナンス性のバランスを考慮。コアロジックをクラスやモジュールに分離せず、1つのスクリプトファイル（`script.ts`）に集約することで、限定的な機能としてのポータビリティを確保。
- **Tailwind CSS**: モダナイズされた「Update」ページのデザインを迅速に横展開し、一貫性のあるユーザー体験を提供するため。
- **localStorage**: サーバーを介さず、ユーザーのブラウザ内のみで採用薬リスト（機密性の高い情報に近いもの）を完結させるため。

## 5. ロジック詳細

### 5.1 フィルタリング順序
1. ウォッチリスト（登録YJコード）による絞り込み（有効時）。
2. クエリ文字列によるテキスト検索。
3. 出荷ステータスによる絞り込み。
4. 成分・区分ごとのグループ集計（サマリー表示用）。

### 5.2 ステータス判定
- `通常`: 文字列に「通常」または「通」を含む。
- `限定`: 文字列に「限定」「制限」「限」「制」のいずれかを含む。
- `停止`: 文字列に「停止」または「停」を含む。

## 6. 今後の展望（ロードマップ）
- 厚生労働省や各メーカーの公開情報の更新頻度向上の反映。
- 共有用URLの短縮機能（多くの品目をパラメータに含める場合の対策）。
- CSV/Excel形式での監視レポート出力機能。
