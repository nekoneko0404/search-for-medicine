# search-for-medicine 収益化および決済手段の比較検討検討報告書

## 1. 収益化アプローチの全体像

「個人情報を取得しない」「包括サブスクと個別サブスクの選択」という要件を満たすため、以下の2つのアプローチを比較検討します。

1. **スマホアプリ化（各アプリストア経由）**
2. **Webアプリのまま運用（Stripe Checkout経由）**

---

## 2. 決済手段ごとの手数料と比較表

各種プラットフォームにおけるサブスクリプション（定期購入）の手数料体系は以下の通りです。

| 決済プラットフォーム | 手数料率（概算） | 備考・適用条件 | 個人情報の取り扱い（開発者側） |
| :--- | :--- | :--- | :--- |
| **Google Play (Android)** | **15%** (または30%) | - 定期購入（サブスクリプション）は初日から**15%**。<br>- 年間収益100万米ドル以下のデベロッパーも15%。それ以降は30%。 | 完全に不要（Googleアカウントに依存）。匿名IDと購入レシートの紐付けのみで実装可能。 |
| **App Store (iOS)** | **15%** (または30%) | - App Store Small Business Programに登録（年間収益100万米ドル以下）で15%。<br>- または定期購入の2年目以降が15%（1年目はプログラム未加入だと30%）。 | 完全に不要（Apple IDに依存）。匿名IDと購入レシートの紐付けのみで実装可能。 |
| **Stripe (Web)** | **約 4.1% 〜 4.4%** | - クレジットカード決済手数料：**3.6%**（Visa, Mastercard等。JCB等は設定により異なる）。<br>- Stripe Billing手数料：**0.5% 〜 0.8%**（定期課金機能の利用）。 | メールアドレス等はStripe側で保持。自社サーバーに保存しない設計（Stripe Checkout上の完結）は可能だが、ユーザーはメールアドレスの入力を求められる。 |

### 手数料に関する考察とリスク検討

* **アプリストア（Google/Apple）のメリット・デメリット:**
  * **メリット:** 手数料（15%）はStripeより高いものの、ユーザー側は「既に登録済みのカードやキャリア決済」でワンタップで決済できるため、**コンバージョン率（CVR）が圧倒的に高くなる**傾向があります。また、一切の個人情報を要求せずに課金が完結します。
  * **デメリット:** PWA（TWA）やCapacitorを利用したアプリ化の手間、および各ストアの審査（特にAppleは医療系・Webラッパーアプリの審査が厳しい傾向）を通過する必要があります。
* **Stripeのメリット・デメリット:**
  * **メリット:** 手数料が圧倒的に安く（約4%台）、Webアプリのままスピーディーに導入できます。ストアの審査に依存しないため、機能追加や価格変更も自由です。
  * **デメリット:** 決済時にユーザーがメールアドレスやクレジットカード情報を入力する手間があり、離脱率が上がります。また「個人情報を一切取得したくない」という要件に対し、自社で保存しなくてもStripe上には情報が残るため、ユーザー視点での「匿名性」はストア決済に劣ります。

## 3. プラン設計アイデア（再掲・詳細化）

### 料金体系（想定）

* **【個別サブスク】 Tier 1**: 月額 200〜300円想定
  * 特定のアプリ（例：小児用量計算のみ）のPro機能をアンロック。
* **【包括サブスク】 Tier 2**: 月額 500〜800円想定
  * 全アプリのPro機能アンロック＋統合検索機能の提供。
* **【法人向け・施設契約】**: 月額 3,000円〜10,000円想定（施設規模による）
  * 法人専用のアクセスコードを発行し、薬局・クリニック内の全スタッフが利用可能。個人アカウント不要。

### アプリ別のPro機能（付加価値）アイデア

#### 💊 小児用量計算（Pediatric Calculator）
* **計算履歴のローカル保存:** 過去の計算履歴を端末内に保存し、ワンタップで再計算。
* **お気に入り処方（マイセット）:** 「体重〇〇kgの風邪セット」などを登録して一括計算。
* **高度な年齢・体重分岐計算の全開放:** アウグスベルガー式や漢方などのフルアクセス。

#### 📦 医薬品供給状況（Supply Status）
* **供給状況の変動通知（ウォッチリスト）:** マークした薬剤の供給状況変化をPush通知。
* **代替薬サジェスト:** 出荷調整時に、同じ効能で通常出荷されている薬を即座に提示。
* **リスト出力機能:** 院内共有用のCSV/PDF出力機能。

#### 🌸 お薬ぱっくん＆花粉アプリ（患者向け）
* **マルチプロファイル（家族アカウント）機能:** スナップショットなど、1端末で複数人のデータをローカル管理。
* **ウィジェット機能:** ホーム画面で花粉飛散量や服薬時間を直接確認（スマホアプリ限定）。
* **服薬カレンダー（PDFレポート）生成:** 医師・薬剤師提出用のレポート作成機能。

#### 🔍 包括サブスク限定機能
* **統合横断検索ダッシュボード:** 一度の検索で「用量」「供給状況」「添付文書」を全アプリ横断で検索できる機能。

## 4. 結論と推奨アプローチ

「個人情報保護（匿名性）」と「ユーザビリティ（課金ハードルの低さ）」を最優先とする場合、**手数料15%を受け入れてでも、主要アプリをPWA（TWA）などでガワネイティブ化し、Google Play経由で配信・課金するアプローチが最も推奨されます。**
これにより、自社システムでは完全に匿名IDのみを扱い、決済プロセスの安全性をGoogleに担保させることができます。

一方で、初期の実装コストを抑えて小さくテストマーケティングを行いたい場合は、Stripeを使ったWeb決済からスモールスタートし、軌道に乗った段階でスマホアプリ化へ移行する戦略も有効です。
