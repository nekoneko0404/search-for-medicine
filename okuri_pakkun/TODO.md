# 開発計画: おくすりぱっくん (Okusuri Pakkun)

小児用服薬支援アプリ「おくすりぱっくん」の開発ロードマップです。

## フェーズ 1: プロジェクト基盤構築
- [x] Next.js プロジェクトの初期化 (v16.1.1)
- [x] Tailwind CSS のセットアップ
- [x] Cloudflare Pages へのデプロイ設定
- [x] 初期ビルドエラーと依存関係の修正 (React 19対応)
- [x] 基本的なユニットテスト環境の構築と修正 (Jest)

## フェーズ 2: データ構造と基本ロジック
- [x] **医薬品マスターデータ (`meds_master.json`) の本格作成**
    - 仕様書に基づき、YJコード、商品名、メーカー、味、相性の良い/悪い食品、根拠を含めたJSON構造を確定する。
    - サンプルデータ（クラリスロマイシン、セフジトレンピボキシルなど代表的な小児薬）を登録する。
- [x] **データアクセスのためのロジック実装**
    - JSONデータを読み込み、検索クエリに基づいて該当する薬剤情報を返すAPIまたは関数を作成する。

## フェーズ 3: UI/UX 実装
- [x] **トップページ（検索画面）の実装**
    - 既存の `drug-navigator` のデザイン言語(Tailwind CSS)を踏襲したシンプルで使いやすい検索バー。
    - メーカーごとの違いを識別できるオートコンプリート機能の実装。
- [x] **詳細ページ（服薬アドバイス表示）の実装**
    - 「味・におい」「相性の良い食品」「相性の悪い食品」「その他注意事項」をカード形式でわかりやすく表示する。
    - エビデンス（情報源）の明記。

## フェーズ 4: PMDAデータ連携 (プロトタイプ範囲)
- [x] PMDA Open Data (XML) の取得・パース処理の調査と実装
    - YJコードをキーにして最新の添付文書情報を取得し、注意事項などを補完する機能。
    - Cloudflare Pages (Static Export) 対応のため、クライアントサイドフェッチ＆パース方式に変更完了。

## フェーズ 5: 仕上げ
- [x] UIのブラッシュアップ
- [x] 追加のテストケース作成とパス確認
- [ ] 本番環境での最終動作確認
